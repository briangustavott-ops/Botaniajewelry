# Setup 02 — N8N

## Requisitos previos

- Docker + Docker Compose instalados (ver [Setup 01](01-docker.md))
- VM accesible en `192.168.137.10`
- Repositorio clonado en `~/botania-repo` (ver [Setup 03](03-sqlite.md))

---

## 1. Crear la estructura de proyecto

```bash
mkdir -p ~/botania/data/n8n
cd ~/botania
```

---

## 2. Crear el archivo `.env`

```bash
nano ~/botania/.env
```

Contenido:

```env
# VM
VM_IP=192.168.137.10

# N8N
N8N_USER=admin
N8N_PASSWORD=cambia_esta_password
N8N_SECURE_COOKIE=false
```

---

## 3. Copiar los archivos de infra desde el repositorio

El `docker-compose.yml` y el `Dockerfile` están versionados en `infra/`:

```bash
cp ~/botania-repo/infra/docker-compose.yml ~/botania/
cp ~/botania-repo/infra/Dockerfile ~/botania/
```

### `docker-compose.yml`

```yaml
services:

  n8n:
    build:
      context: .
      dockerfile: Dockerfile
    image: botania/n8n:local
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=${N8N_USER}
      - N8N_BASIC_AUTH_PASSWORD=${N8N_PASSWORD}
      - N8N_HOST=${VM_IP}
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - WEBHOOK_URL=http://${VM_IP}:5678/
      - GENERIC_TIMEZONE=Europe/Madrid
      - TZ=Europe/Madrid
      - N8N_SECURE_COOKIE=${N8N_SECURE_COOKIE}
      - N8N_CUSTOM_EXTENSIONS=/n8n-custom-nodes
    volumes:
      - ./data/n8n:/home/node/.n8n
      - ./data/db:/data/db
```

### `Dockerfile` — multi-stage build

La imagen oficial `n8nio/n8n` usa **Docker Hardened Images (DHI)**: Alpine sin gestor de paquetes (`apk` ni `apt-get` disponibles). Esto impide instalar herramientas de compilación directamente, lo que hace fallar los nodos de comunidad con módulos nativos (como `sqlite3`).

Solución: **multi-stage build** — compilamos en una imagen Alpine completa y copiamos el resultado a la imagen endurecida.

```dockerfile
# Stage 1: compilar sqlite3 nativo en node:24-alpine (tiene apk)
FROM node:24-alpine AS builder

RUN apk add --no-cache python3 make g++
WORKDIR /build
RUN npm install n8n-nodes-sqlite

# Stage 2: imagen N8N endurecida con el nodo ya compilado
FROM n8nio/n8n:latest

USER root

# /n8n-custom-nodes NO está cubierto por el volumen ./data/n8n
# → los módulos del build siempre están disponibles
RUN mkdir -p /n8n-custom-nodes/node_modules
COPY --from=builder /build/node_modules /n8n-custom-nodes/node_modules

USER node
```

> **`N8N_CUSTOM_EXTENSIONS=/n8n-custom-nodes`** indica a N8N que cargue nodos adicionales desde ese directorio al arrancar.

> **`N8N_SECURE_COOKIE=false`** es necesario porque el acceso es por HTTP local (sin TLS).

---

## 4. Construir la imagen y arrancar N8N

```bash
cd ~/botania

# Construir imagen personalizada — tarda unos minutos la primera vez
# (descarga node:24-alpine, compila sqlite3, construye imagen final)
docker compose build

# Arrancar en background
docker compose up -d
```

Verificar que el contenedor está corriendo:

```bash
docker compose ps
docker compose logs n8n
```

---

## 5. Verificar acceso desde Windows

Abrir en el navegador de Windows:

```
http://192.168.137.10:5678
```

Debe aparecer la pantalla de login de N8N. Usar las credenciales del `.env`.

---

## 6. Nodo SQLite — disponible sin instalación

El nodo `n8n-nodes-sqlite` está **precompilado e incluido** en la imagen mediante el multi-stage build. No es necesario instalarlo desde la UI de N8N.

Al crear un workflow, el nodo **SQLite** aparece directamente en el catálogo de nodos.

> Si en el futuro se necesita añadir otro nodo de comunidad con módulos nativos, basta con añadir `npm install <paquete>` en el Stage 1 del Dockerfile y reconstruir la imagen.

---

## 7. Comandos útiles

```bash
# Ver logs en tiempo real
docker compose logs -f n8n

# Parar N8N
docker compose stop

# Arrancar N8N
docker compose start

# Reiniciar
docker compose restart n8n

# Actualizar N8N (rebuilding con la nueva imagen base)
docker compose build --no-cache && docker compose up -d
```
