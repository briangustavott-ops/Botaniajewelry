# Setup 02 — N8N

## Requisitos previos

- Docker + Docker Compose instalados (ver [Setup 01](01-docker.md))
- VM accesible en `192.168.137.10`

---

## 1. Crear la estructura de proyecto

```bash
mkdir -p ~/botania/data/n8n
cd ~/botania
```

---

## 2. Crear el archivo `.env`

```bash
nano ~/botania/.env
```

Contenido:

```env
# VM
VM_IP=192.168.137.10

# N8N
N8N_USER=admin
N8N_PASSWORD=cambia_esta_password
N8N_SECURE_COOKIE=false
```

---

## 3. Crear `docker-compose.yml`

```bash
nano ~/botania/docker-compose.yml
```

Contenido:

```yaml
services:

  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=${N8N_USER}
      - N8N_BASIC_AUTH_PASSWORD=${N8N_PASSWORD}
      - N8N_HOST=${VM_IP}
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - WEBHOOK_URL=http://${VM_IP}:5678/
      - GENERIC_TIMEZONE=Europe/Madrid
      - TZ=Europe/Madrid
      - N8N_SECURE_COOKIE=false
    volumes:
      - ./data/n8n:/home/node/.n8n
```

> **Nota:** `N8N_SECURE_COOKIE=false` es necesario porque el acceso es por HTTP local (sin TLS). En un entorno con HTTPS no sería necesario.

---

## 4. Arrancar N8N

```bash
cd ~/botania
docker compose up -d
```

Verificar que el contenedor está corriendo:

```bash
docker compose ps
docker compose logs n8n
```

---

## 5. Verificar acceso desde Windows

Abrir en el navegador de Windows:

```
http://192.168.137.10:5678
```

Debe aparecer la pantalla de login de N8N. Usar las credenciales del `.env`.

---

## 6. Comandos útiles

```bash
# Ver logs en tiempo real
docker compose logs -f n8n

# Parar N8N
docker compose stop

# Arrancar N8N
docker compose start

# Reiniciar
docker compose restart n8n

# Actualizar imagen de N8N
docker compose pull && docker compose up -d
```

---

## Resultado

| Servicio | URL | Credenciales |
|---------|-----|--------------|
| N8N | `http://192.168.137.10:5678` | admin / [password del .env] |

Los datos de N8N (workflows, credenciales) se guardan en `~/botania/data/n8n/` — basta con copiar esa carpeta para hacer backup.
